# E-Bora — Telegram Mini App (техника Dyson)

Каталог Dyson с корзиной и оформлением заказа через менеджера
**@eborauz_Admin**. Языки RU / UZ: автоопределение по языку Telegram
+ переключатель в шапке.

## Файлы
```
index.html          — всё приложение (один файл, без сборки)
bot.py              — бот: /start присылает кнопку, открывающую каталог
download_images.sh  — скачивает фото каталога в images/ (рекомендуется)
requirements.txt    — зависимость для bot.py
```

## Шаг 1. Деплой каталога (GitHub Pages)
1. Создайте репозиторий, залейте файлы (`index.html` в корень).
2. Рекомендуется: локально `bash download_images.sh` и закоммитить
   папку `images/` — фото перестанут зависеть от стороннего CDN.
3. Settings → Pages → Source: branch `main`, folder `/ (root)` → Save.
4. Через минуту каталог доступен на `https://<username>.github.io/<repo>/`.

## Шаг 2. Кнопка Start в боте
Вариант А (без сервера): в @BotFather → `/setmenubutton` → выбрать бота →
вставить URL из шага 1 → подпись, например `Каталог`. Кнопка меню (рядом
с полем ввода) будет открывать mini app.

Вариант Б (кнопка прямо в ответ на /start): запустить `bot.py`.
1. `pip install -r requirements.txt`
2. В `bot.py` вписать `TOKEN` (из @BotFather) и `WEBAPP_URL` (из шага 1).
3. `python3 bot.py` — держать запущенным на любом сервере
   (Railway / VPS / даже Termux).
По `/start` бот пришлёт кнопку «🛍 Каталог / Katalog», открывающую
приложение. Лучший вариант — сделать и А, и Б.

## Настройка каталога
Всё в `index.html`:
- `CONFIG.managerUser` — username менеджера без @ (сейчас `eborauz_Admin`);
- `PRODUCTS` — товары. `price: null` = «цена по запросу»; впишите число,
  чтобы показать цену и включить итог. `spec: {ru, uz}` — характеристики.

## Как уходит заказ
Кнопка «Оформить у менеджера» открывает чат @eborauz_Admin с готовым
текстом заказа (позиции, количество, сумма). Текст дополнительно
копируется в буфер на случай, если клиент Telegram не подставит его сам.
